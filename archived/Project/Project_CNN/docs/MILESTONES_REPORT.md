### Custom test (`m1_selfcheck.sh`)

What it does:
- Builds an **expanded batch sweep** beyond the public 100/1000/10000:
  `GPU_BATCHES = (100, 1000, 4095, 4096, 8767, 10000)` — covers tile-boundary cases (4095 vs 4096) and the previously-failing 8767/10000.
- For each batch it runs `./m1_gpu N`, parses `Test Accuracy:` and sums all `Op Time:` lines.
- Public batches (100/1000/10000) are checked against the expected accuracy with `±0.002` tolerance; edge batches must be `≥ 0.80`.
- B=10000 must have total Op Time `≤ 300 ms`.
- Warns if Op Time sum `< 0.1 ms` (silent kernel-launch failure).

How to run:
```bash
sbatch m1_selfcheck.slurm        # default GPU sweep
bash ./m1_selfcheck.sh --cpu     # add CPU sweep
ACCURACY_TOL=0.001 bash ./m1_selfcheck.sh   # tighten tolerance
```

## Milestone 2 — Profiling, Unrolling, Kernel Fusion

### What was done

1. **CPU profiling** with `gprof` (rebuild with `-pg` via `build_pg.sh`, B=1000 run).
2. **Implemented matrix unrolling kernel** + host wiring for the unfused unroll-then-matmul-then-permute pipeline.
3. **Implemented fused kernel** (`matmul_conv_fused`) that streams unroll into shared-memory tile loads and writes the permuted output directly.
4. **Profiled three configurations** (`m1_gpu`, `m2_unroll`, `m2_fused`) with `nsys` and `ncu`.
5. **Sanitizer runs** (`compute-sanitizer`) on both unroll and fused at B=100 — both report `0 errors`.
6. Extended the M1 selfcheck pattern into `m2_selfcheck.sh` for both binaries.

### Code files modified

| File | Purpose |
|---|---|
| [project/src/layer/custom/unroll-new-forward.cu](project/src/layer/custom/unroll-new-forward.cu) | `matrix_unrolling_kernel` + prolog/`conv_forward_gpu`/epilog. Uses `size_t` indexing because B=10000 unrolled matrix exceeds `INT_MAX`. |
| [project/src/layer/custom/kernel-fusion-forward.cu](project/src/layer/custom/kernel-fusion-forward.cu) | `matmul_conv_fused` tiled GEMM that loads input via implicit unroll and writes already-permuted output. |
